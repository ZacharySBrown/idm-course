# Commit the pre-switch section before you flip chains

- **CPU problem (callback from W8):** eight-chain Racks with three reverbs each stutter once you stack a second full arrangement on top. Freeze works but blocks Session A/B auditioning.
- **The four-step discipline:**
  1. Dial in the pre-switch section (bars 1–32). Best version of itself.
  2. **Bounce Groups** (Live 12.3) or Freeze + Flatten. `bib:ableton_live_12_manual`, *Freeze and Flatten*.
  3. Delete or Activate-disable source tracks. Save as `v02`, never overwrite.
  4. Build the post-switch chain with the CPU you just recovered.
- This is the Burial Bounce-and-Arrange half of the workflow (`bib:burial_soundforge_quote`, Zak-Handbook §8).
- **Honest cost:** committed stems are recall-hostile. Tweaking a reverb tail means reopening the old set and re-rendering. The rule is *commit, then switch* — not *switch, then commit*.
- Bounce when you are bored with the pre-switch, not when you are still excited by it. Rick Rubin's rule, `bib:rick_rubin_creative_act`.
