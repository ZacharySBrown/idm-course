# NTS Sessions (2018) — versioning as practice

Eight hours of music across four 2-hour broadcasts. Described in press and WATMM consensus as "a persistent Max/MSP environment" — the same patch family reworked across years.

Booth, widely quoted from the 2018 press cycle (flag — attributed via multiple Pitchfork + forum transcriptions; we list the interview in `needs_bib`): *"I think the oldest thing is from 2011… weird recent jams using old patches."*

The practice: **archival, version-controlled composition.** Save many states. Revisit years later. Treat a Max patch the way a software engineer treats a repo.

In Ableton, this maps to:

- Save Live sets with version numbers. v01, v02, v03.
- Use **Chain Selector + Macro Variations** (12.2+) to store 8–16 recallable patch snapshots inside one rack.
- When a sound idea works, save it as a rack preset with a dated name. Raid the preset folder six months later.
