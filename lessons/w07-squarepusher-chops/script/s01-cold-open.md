# s01 — Cold open

The thing about Squarepusher is he does not like you — or anyone — watching the screen while he works. [pause 300ms] That is the whole plot.

Tom Jenkinson, to Vijith Assar in *Tape Op* eighty-nine, twenty-twelve, page thirty-four, and I quote: *"When the graphical information is too vivid, it makes it harder to retain the information in my memory."* [pause 400ms]

He sequences on a Yamaha QY700. It is a 1997 hardware sequencer. It has a 320 by 240 monochrome LCD the size of a postcard. You enter notes by number. There is no piano roll. [pause 300ms] He chose it on purpose.

This is not gear-nostalgia. It is a working principle. The Jenkinson premise is that a vivid screen pollutes the mental model of what is happening in the studio. If you can see the piano roll, you start *fixing* the piano roll. If all you see is a number, you run the sequencer and listen.

So. [pause 250ms] This week we steal from that posture. We are not going to buy a QY700. We are going to chop an Amen break in Ableton, which takes one right-click, and then we are going to put constraints on ourselves that simulate the refusal. Single pass. No undo. Commit what you get.

The source material this week is the `w07-funky-fanfare-amen` render. Zak's Stemforge ran Funky Fanfare through the `idm_crushed` pipeline. You will get a drums stem and a folder of sixteenth-note chops. We slice the drums stem into a Drum Rack, play a pattern once, bounce, ship. [pause 300ms]

Everything else on this lesson is the receipts for why that exercise exists. ~Go Plastic, Hard Normal Daddy, Big Loada, The Metallurgist.~ All of it points to the same discipline: the sequencer is a musical instrument, not a text editor.

One last thing before we move. The QY700 is *not* the only anti-computer posture in the course. We will hit a side-note in a minute about Kevin Parker's Boss BR-1600. File that for later. [pause 200ms] First, the brain of the Go Plastic rig.
