# Self-master or send it out — the Fridmann test

The decision is not about skill. It is about *finishing*.

A line circulates in production forums, attributed to Dave Fridmann: *"if it's not finished, it's not mastered."* <Citation bib="fridmann_davefridmann_com" /> I could not resolve the verbatim source in either the Handbook or the Compendium — treat the wording as community paraphrase that aligns with Fridmann's public philosophy, flagged in `needs_bib:`. The meaning holds either way.

The point: mastering is not a process that rescues an unfinished mix. It is a translation pass applied to a mix that is already decided. If the low end is not committed, a mastering engineer cannot fix it. If the arrangement has four options and you have not chosen, no amount of Pro-L 2 will choose for you.

The test is one question, honest: **do you know what the track is supposed to sound like?** If yes, self-master this week. Use the tools above. Translate. Move on.

If no — send it to an engineer *after* you finish the mix. Not before. Not instead of. An engineer costs money. An unfinished mix costs more because it will not finish at their desk either.

For the twelve-week course, self-master. Next year, decide per track.
