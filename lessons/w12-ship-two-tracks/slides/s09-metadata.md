# Metadata hygiene — ISRC, credits, artwork, file naming

The boring slide. Boring wins.

- **ISRC** — International Standard Recording Code. Twelve characters, allocated per recording. Distributors allocate automatically. If you are Bandcamp-only, you can skip this initial release and add ISRCs when you move to a DSP distributor. The RIAA US ISRC agency page is the canonical reference; flagged as `needs_bib:`.
- **Composer credits** — your legal name or a consistent pseudonym. This is what registers with PROs (ASCAP, BMI, SOCAN). Inconsistent credits across twelve releases means inconsistent royalty collection. Pick one. Write it the same way every time.
- **Genre tags** — Bandcamp accepts multiple. "IDM" and "Electronic" plus one sub-tag ("glitch," "ambient," "breakbeat") is enough. Searchability comes from accuracy, not from stuffing.
- **Artwork dimensions** — 3000 × 3000 px, JPEG or PNG, under 10 MB. This is the DSP-common minimum. Lower resolution will upload; it will look worse on every platform.

File naming convention worth committing to: `artist_trackname_v01_mixprint.wav`, then `v02`, `v03`. When you export a new master, version it and leave the old one. Do not overwrite. Do not use emoji. Do not use exclamation marks in track titles; some distributors silently strip them. Lowercase is the safer default. <Citation bib="edmprod_intermediate" />
