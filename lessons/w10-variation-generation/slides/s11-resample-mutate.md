# The Resample-and-Mutate engine — Mr. Bill's forcing function

Spec A §5.6. Mr. Bill describes this as a core loop, not a technique. <Citation bib="mr_bill_resample_tutorial" />

**Every 8–16 bars:**

1. Route drum bus (or full mix) to a new audio track — **Audio From → Resampling**, or the bus.
2. Record 4–8 bars.
3. **Delete the source MIDI.**
4. Chop into slices. Drum Rack Slice-to-Transients, or Simpler Slicing.
5. Pitch-shift, reverse, bit-crush a few. B and C section material.

<Citation bib="ableton_live_12_manual" />

**Why delete.** Frozen audio is psychologically expensive to un-freeze. That expense is the feature.

**Sample Offset envelope.** Draw ±8 sixteenth-notes on the printed loop. New patterns from the same audio — MPC-style beat repeat.

**Consolidate every 8–16 bars.** Cmd+J. The bounce is the arrangement move.
