# Why delete — the anti-perfectionism machine

"Bounce and delete" sounds like a loss of power. It is a loss of *optionality*, which is a different thing.

Undo is a procrastination engine. If your kick is one Cmd+Z away, you will spend thirty minutes flipping between versions. Print the kick bus, delete the source, and the decision is made. The only remaining move is *forward*. <Citation bib="mr_bill_resample_tutorial" />

Rick Rubin, *The Creative Act* — "The goal is to be *done*, not *perfect*." <Citation bib="rick_rubin_creative_act" /> The bounce is how you operationalize that sentence.

Eno, Oblique Strategies — "Discard an axiom." <Citation bib="oblique_strategies_eno" /> Same thing, phrased as a card.

**The observable failure mode** this prevents — spending ninety minutes tweaking a hi-hat inside a loop that has been finished for the first eight of those minutes. The loop is fine. You were in a rabbit hole.

**One caveat, important.** Versioning discipline (s12) is the safety net. Bounce and delete *within* the Live set. Save the set timestamped *outside* the Live set. The two disciplines hold each other up.
