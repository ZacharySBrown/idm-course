## Annotated tour — the patching environment

AudioMulch is not a DAW. It is a node-graph for live audio.

- **Canvas** — drag-and-drop "Contraptions" (AudioMulch's term for modules) onto a work surface, connect outputs to inputs with patch cables.
- **Contraption types** — file player, live audio input, filter, granulator, delay, bit-crusher, reverb, gate, sequencer, routing-splitter, mixer, output.
- **Metasurface** — XY control page that interpolates between named "scenes" of all module parameters. Morph between presets in real time with a mouse drag.
- No tracks, no timeline. The audio is a stream through the graph. You record the output stream as the "take."

The closest modern analog is Max/MSP or Bidule. Unlike them, AudioMulch is single-window, single-canvas, opinionated — and that opinion shaped a generation of IDM sample mangling. `bib:audiomulch_docs`.
