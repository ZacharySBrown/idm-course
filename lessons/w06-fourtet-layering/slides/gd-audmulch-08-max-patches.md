## Max for Live as the truer parallel

For a 1:1 paradigm match, Max for Live is closer than any native Ableton device.

- **Max for Live** — embed a Max/MSP patch as an Ableton device. Patch graph *is* the instrument, same logic as AudioMulch.
- **Community M4L devices** in the AudioMulch spirit:
  - *Granulator III* (Henke, free) — granular playback of a sample, XY morph surface. Closest Metasurface-style gesture control in the Live ecosystem. `bib:henke_granulator_iii`.
  - *Buffer Shuffler* / *Convolution Reverb* / *Granulator* — all ship in Max for Live suite.
  - Any Max patch you design yourself with `buffer~`, `sfplay~`, `gen~`, and Metasurface-style preset morph via `pattr` + `pattrstorage`.

The paradigm survives in M4L. The specific early-2000s IDM flavor of AudioMulch does not — partly because M4L can do more, partly because the ergonomics of a single-window crash-prone Windows XP program are, on reflection, not worth recreating.
