## Ableton bridge — Wavetable + Operator running presets

No native Omnisphere substitute, but the *discipline* ports with care.

| Omnisphere move | Ableton equivalent |
|---|---|
| Browse alphabetically, commit | Wavetable factory library → sort by name, commit |
| 3-layer pre-stacked patch | Instrument Rack with 3 chains, one Wavetable preset per chain |
| FX chain baked into patch | Audio Effect Rack on the instrument track, saved as default |
| "Plectra Sub"-style plucked sub bass | Operator preset *Bass > Plucky Sub*, unedited |
| Orb modulation | Macro 1 + Macro 2 on the rack, assigned pre-save |

The hard rule: **open the browser, commit, close the browser**. If you edit a parameter you have broken the experiment. Drag in a Wavetable preset, play it, move on.

Anti-pattern: producers who load Wavetable, spend 40 minutes sculpting a patch, and then build the track around it. That is the rabbit hole this lesson is immunizing against.
