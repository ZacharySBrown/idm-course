## Algorithm continued — what the engine cannot do

Every synth has limits. Naming Omnisphere's limits defends against the "everything machine" framing.

- **CPU** — stacked layers + Aux FX eat cores fast. Hebden's laptop-only workflow implies freezing patches to audio early. Not discussed on `Tape Notes #140`, but any laptop producer doing 40-track sessions is printing.
- **Sample library is fixed** — no user soundsource import on v2.x without paid expansion packs. For a 2003-Hebden who *was* all about imported source material, Omnisphere would have been a prison.
- **No modular routing** — the signal path is fixed per the slide-06 diagram. Contrast AudioMulch (next module), where routing *is* the instrument.
- **Preset-first by design** — the oscillator page exists but is clearly an afterthought compared to the browser. The box *wants* you to not tweak.

The constraint is load-bearing. Omnisphere is the right instrument for 2022-Hebden precisely because it blocks rabbit holes.
