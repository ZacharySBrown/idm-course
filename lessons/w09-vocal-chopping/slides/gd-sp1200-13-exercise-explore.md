# Exercise B — the 2.5-second constraint, self-imposed

Goal: make a 4-bar beat where every sampled element is **≤ 2.5 seconds**. Prove to yourself that the constraint shapes the beat, not the chain.

Steps:

1. Fresh project. Open a Drum Rack. Nothing loaded yet.
2. Pick 8 source samples — vocal chops, drum breaks, bass hits, whatever. Each must be **trimmed to 2.5 seconds or less** in the Simpler start/end markers. Longer source material has to be chopped, not looped.
3. Program a 4-bar pattern using only these 8 samples.
4. No stretched loops. No 8-bar phrases. If you can't fit an idea in 2.5 seconds, you chop it or cut it.
5. Bounce as `hp_2p5sec.wav`.
6. For comparison: redo the same musical idea with no length limit. Bounce as `hp_unlimited.wav`.

Deliverable: two bounces, plus one paragraph naming the specific musical decisions the 2.5-second limit forced — usually: more chops, shorter phrases, more retriggers, more dense patterns. This constraint is what made *Enter the Wu-Tang* sound different from *Illmatic* sound different from *Fantastic Vol. 1*. Same producers, same records, same limit, different outcomes (`bib:charnas_dilla_time`).

Reflect: constraints beat chains every time.
